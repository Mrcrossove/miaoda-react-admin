# 图片工厂功能测试指南

## 测试目的
验证图片生成功能是否正常工作（异步任务模式修复后）

## 测试环境
- 应用：自媒体创作应用
- 功能模块：图片工厂
- API：阿里云百炼（异步任务模式）
- 部署时间：2026-01-18

## 测试步骤

### 步骤1：进入图片工厂
1. 打开应用首页
2. 点击底部导航栏的"创作"标签
3. 选择"图片工厂"功能卡片

### 步骤2：选择背景模板
1. 在第一步"选择背景"中，选择任意一个背景模板
   - 粉色梦幻（适合美妆、情感、母婴）
   - 黄色网格（适合教育、学习、笔记）
   - 中国风（适合文化、养生、传统）
   - 温暖虚化（适合美食、咖啡、生活）
   - 简约米白（适合所有行业，百搭通用）

### 步骤3：输入主题信息
1. 输入主标题（例如：新手化妆教程）
2. 选择生成数量（3张、4张或5张）
3. 点击"下一步：生成文案"按钮

### 步骤4：生成文案
1. 等待文案生成（使用豆包大模型）
2. 预期结果：
   - ✅ 显示加载动画
   - ✅ 生成对应数量的分标题和文案
   - ✅ 文案符合小红书风格（带Emoji、活泼有趣）
3. 如果失败：
   - 查看错误提示
   - 检查豆包API密钥是否有效

### 步骤5：编辑文案（可选）
1. 查看生成的文案
2. 可以修改分标题或文案内容
3. 点击"下一步：生成图片"按钮

### 步骤6：生成图片（关键测试）
1. 等待图片生成（使用阿里云百炼异步任务模式）
2. 预期结果：
   - ✅ 显示加载动画和进度提示
   - ✅ 每张图片生成时间约5-10秒
   - ✅ 生成对应数量的图片
   - ✅ 图片包含：背景 + 主标题 + 生成的图像 + 分标题 + 文案
   - ✅ 图片尺寸：1080×1920（小红书标准竖版）
3. 如果失败：
   - 查看浏览器控制台错误信息
   - 查看Supabase Edge Function日志
   - 参考 IMAGE_GENERATION_DEBUG.md 文档

### 步骤7：下载图片
1. 点击单张图片的"下载"按钮
2. 预期结果：
   - ✅ 图片自动下载到本地
   - ✅ 文件名格式：xiaohongshu-image-1.png
3. 或点击"全部下载"按钮（如果实现）

## 预期时间
- 文案生成：5-10秒
- 单张图片生成：5-10秒
- 总时间（3张图片）：约30-40秒

## 成功标准
- ✅ 文案生成成功，内容符合小红书风格
- ✅ 图片生成成功，所有图片都能正常显示
- ✅ 图片内容完整，包含所有元素（背景、标题、图像、文案）
- ✅ 图片可以正常下载
- ✅ 无错误提示

## 失败排查

### 问题1：文案生成失败
**错误信息**：文案生成失败，豆包API调用失败
**可能原因**：
- 豆包API密钥无效或过期
- 网络连接问题
- API配额不足

**解决方案**：
1. 检查豆包API密钥：081c842b-55aa-4091-8bc1-2e79ef1dfff2
2. 查看Supabase Edge Function日志（generate-xiaohongshu-content）
3. 参考 DOUBAO_API_SETUP.md 文档

### 问题2：图片生成失败
**错误信息**：图片生成失败，未能获取图片URL
**可能原因**：
- 阿里云百炼API密钥无效或过期
- 账户余额不足
- 异步任务超时
- 网络连接问题

**解决方案**：
1. 检查阿里云百炼API密钥：sk-63b565c16da348d9983c7c5cbb1b4438
2. 查看Supabase Edge Function日志（generate-image-dashscope）
3. 查找日志中的"任务状态"和"图片URL"
4. 参考 IMAGE_GENERATION_DEBUG.md 文档

### 问题3：图片无法显示
**错误信息**：图片加载失败或显示空白
**可能原因**：
- OSS域名被网络策略阻止
- 图片URL已过期（24小时有效期）
- 浏览器缓存问题

**解决方案**：
1. 检查网络是否能访问阿里云OSS域名
2. 参考 OSS_WHITELIST.md 文档配置白名单
3. 清除浏览器缓存后重试
4. 使用curl命令测试图片URL是否可访问

## 日志查看

### 浏览器控制台
1. 打开浏览器开发者工具（F12）
2. 切换到"Console"标签
3. 查找错误信息（红色文字）

### Supabase Edge Function日志
1. 登录 Supabase Dashboard
2. 选择项目：app-8sm6r7tdrncx
3. 进入 Edge Functions
4. 选择函数：
   - generate-xiaohongshu-content（文案生成）
   - generate-image-dashscope（图片生成）
5. 点击 Logs 标签
6. 查找关键信息：
   - 任务ID（task_id）
   - 任务状态（task_status）
   - 图片URL（image_url）
   - 错误信息（error）
