# AI二创功能测试指南

## 问题修复说明

### 问题描述
- AI创作显示成功，但标题和文案框没有显示内容
- 浏览器控制台显示"解析数据失败"

### 根本原因
Edge Function对文心大模型API返回的SSE流式数据进行了二次包装，导致前端收到双重包装的数据格式，JSON解析失败。

### 修复方案
- Edge Function直接透传API的response.body，不做二次包装
- 保持原始SSE格式不变
- 前端添加详细的调试日志

## 常见问题排查

### 问题1：控制台显示"解析数据失败"

**可能原因**：
- 数据格式不正确
- JSON解析失败

**排查步骤**：
1. 查看"收到原始数据"日志，检查数据格式
2. 查看"解析后的数据"日志，确认是否成功解析
3. 查看"错误堆栈"日志，定位具体错误位置

**预期数据格式**：
```json
{
  "id": "as-xxx",
  "choices": [
    {
      "index": 0,
      "delta": {
        "role": "assistant",
        "content": "标"
      },
      "finish_reason": null
    }
  ]
}
```

### 问题2：标题和文案框没有内容

**可能原因**：
- 正则匹配失败
- AI返回的格式不符合预期

**排查步骤**：
1. 查看"累积内容"日志，确认是否接收到完整内容
2. 查看"匹配到标题"和"匹配到文案"日志，确认是否成功匹配
3. 检查AI返回的内容格式

**预期内容格式**：
```
标题：✨超好用的美妆神器！

文案：
姐妹们，今天给大家分享一款超级好用的美妆神器...
```

### 问题3：生成速度很慢

**可能原因**：
- 网络延迟
- API服务器负载高

**解决方案**：
- 等待完成（通常10-30秒）
- 如果超过1分钟，刷新页面重试

### 问题4：生成内容不完整

**可能原因**：
- 网络中断
- API返回被截断

**解决方案**：
- 再次点击"AI智能二创"按钮重新生成
- 检查网络连接

## 数据流程图

```
用户点击"AI智能二创"
    ↓
前端发送请求到Edge Function
    ↓
Edge Function调用文心大模型API
    ↓
API返回SSE流式数据
    ↓
Edge Function直接透传response.body
    ↓
前端SSE Hook接收数据
    ↓
eventsource-parser解析SSE格式
    ↓
提取data字段（JSON字符串）
    ↓
JSON.parse解析为对象
    ↓
提取choices[0].delta.content
    ↓
累积到fullContent
    ↓
正则匹配提取标题和文案
    ↓
更新到输入框
    ↓
生成完成
```

## 性能指标

### 正常情况

- **首字响应**：< 2秒
- **完整生成**：10-30秒
- **数据包数量**：50-200个
- **每个数据包大小**：50-200字节

### 异常情况

- **首字响应**：> 5秒 → 网络延迟或API响应慢
- **完整生成**：> 60秒 → API超时或网络问题
- **数据包数量**：< 10个 → 生成内容过短或中断
- **每个数据包大小**：> 1KB → 数据格式异常

## 成功标准

✅ **功能正常的标志**：

1. 控制台没有"解析数据失败"错误
2. 能看到"收到原始数据"、"解析后的数据"、"提取的chunk"等日志
3. 能看到"匹配到标题"和"匹配到文案"日志
4. 标题输入框实时显示生成的标题
5. 文案输入框实时显示生成的文案
6. 生成完成后显示"AI二创完成！"提示
7. 标题长度 ≤ 20字
8. 文案长度 ≤ 200字
