# AI文案内容过滤系统测试文档

## 功能概述

AI文案内容过滤系统用于自动移除生成文案中的标题、正文等标识文字，保留实际内容，确保文案简洁美观。

## 过滤规则

### 1. 括号标识过滤
- `【标题】` → 移除
- `【正文】` → 移除
- `【内容】` → 移除
- `【主题】` → 移除
- `【题目】` → 移除
- `【主旨】` → 移除
- `【详情】` → 移除
- `【描述】` → 移除
- `[标题]` → 移除（方括号）
- `[正文]` → 移除（方括号）

### 2. 冒号前缀过滤
- `标题：` → 移除
- `标题:` → 移除（英文冒号）
- `正文：` → 移除
- `内容：` → 移除
- `主题：` → 移除

### 3. 破折号前缀过滤
- `标题-` → 移除
- `标题－` → 移除（全角）
- `标题—` → 移除（长破折号）
- `正文-` → 移除

### 4. 数字序号模式过滤
- `1.标题：` → 移除
- `2.正文：` → 移除
- `1、标题：` → 移除

### 5. 单独标识词过滤
- 行首的"标题\n" → 移除
- 行首的"正文\n" → 移除

### 6. 格式清理
- 超过2个连续换行 → 保留2个
- 首尾空白字符 → 移除

## 技术实现

### 核心函数

#### filterContentLabels(content: string): string
过滤文案中的所有标识文字，返回清理后的内容。

#### filterTitleLabels(title: string): string
专门过滤标题相关标识。

#### filterBodyLabels(body: string): string
专门过滤正文相关标识。

#### filterContentList<T>(items: T[]): T[]
批量过滤文案列表。

#### hasContentLabels(content: string): boolean
检查文案是否包含标识文字。

#### getFilterStats(original: string, filtered: string)
获取过滤统计信息。

### 正则表达式说明

```typescript
// 括号标识
/【标题】\s*/g  // 匹配【标题】及后续空白

// 冒号前缀（行首）
/^标题[：:]\s*/gm  // 匹配行首的"标题："或"标题:"

// 破折号前缀（行首）
/^标题[-－—]\s*/gm  // 匹配行首的"标题-"等

// 数字序号
/^\d+[.、]\s*标题[：:]\s*/gm  // 匹配"1.标题："等

// 单独标识词
/^标题\s*\n/gm  // 匹配行首的"标题"后跟换行
```

## 用户体验优化

### 优化前
用户需要：
1. 生成文案
2. 手动删除【标题】、【正文】等标识
3. 调整格式
4. 复制发布

### 优化后
用户只需：
1. 生成文案
2. 直接复制发布

**节省时间**：每次生成文案节省 30-60 秒的手动编辑时间

## 注意事项

1. **不影响正常内容**：
   - 品牌名中的括号内容（如【XX品牌】）不会被误删
   - 正文中的数字列表不会被误删
   - 只删除行首或特定位置的标识

2. **保持格式**：
   - 保留原文案的换行
   - 保留emoji表情
   - 保留正常的标点符号

3. **智能清理**：
   - 自动清理多余空行
   - 自动清理首尾空白
   - 保持文案整洁

## 扩展性

如需添加新的过滤规则，只需在 `src/utils/contentFilter.ts` 中添加相应的正则表达式模式即可。

例如，添加新的标识词：
```typescript
const newPatterns = [
  /【新标识】\s*/g,
  /^新标识[：:]\s*/gm,
];
```
