# Sora 2 视频生成 API 文档

## 概述

本文档描述了项目中使用的 Sora 2 异步视频生成 API 的调用方式。

## API 端点

```
https://api.apiyi.com/v1/videos
```

## 认证方式

**重要**：API Key 需要在请求头中传递。

```
Authorization: YOUR_API_KEY
```

> 注意：根据 API 文档示例，部分情况下不需要 `Bearer` 前缀。

## 完整流程（三步）

### 第一步：提交视频生成请求

**请求方式**: `POST`

**Content-Type**: `multipart/form-data`

**请求参数**:

| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| `model` | string | ✅ | 固定值：`sora-2` |
| `prompt` | string | ✅ | 视频描述，建议详细描述场景、动作、风格 |
| `size` | string | ✅ | `720x1280`（竖屏）或 `1280x720`（横屏） |
| `seconds` | string | ✅ | 视频时长：`10` 或 `15` |
| `input_reference` | file | ❌ | 参考图片（PNG/JPG/JPEG），用于图生视频 |

**请求示例**:

```bash
curl -X POST "https://api.apiyi.com/v1/videos" \
  -H "Content-Type: multipart/form-data" \
  -H "Authorization: YOUR_API_KEY" \
  -F "model=sora-2" \
  -F "prompt=电商宣传视频，展示产品细节" \
  -F "size=720x1280" \
  -F "seconds=10"
```

**响应示例**:

```json
{
  "id": "video_abc123def456",
  "status": "submitted",
  "progress": 0,
  "prompt": "电商宣传视频，展示产品细节",
  "model": "sora-2",
  "size": "720x1280",
  "seconds": "10",
  "created_at": 1704067200,
  "url": null,
  "completed_at": null
}
```

### 第二步：轮询查询视频状态

**请求方式**: `GET`

**端点**: `/v1/videos/{video_id}`

**请求示例**:

```bash
curl -X GET "https://api.apiyi.com/v1/videos/video_abc123def456" \
  -H "Authorization: YOUR_API_KEY"
```

**响应示例（生成中）**:

```json
{
  "id": "video_abc123def456",
  "status": "in_progress",
  "progress": 45,
  "url": null
}
```

**响应示例（完成）**:

```json
{
  "id": "video_abc123def456",
  "status": "completed",
  "progress": 100,
  "url": "https://api.apiyi.com/v1/videos/video_abc123def456/content"
}
```

**状态说明**:

| 状态 | 说明 |
|------|------|
| `submitted` | 请求已提交，等待处理 |
| `in_progress` | 视频生成中 |
| `completed` | 生成完成，可下载 |
| `failed` | 生成失败 |

### 第三步：下载视频

**请求方式**: `GET`

**端点**: `/v1/videos/{video_id}/content`

**请求示例**:

```bash
curl -X GET "https://api.apiyi.com/v1/videos/video_abc123def456/content" \
  -H "Authorization: YOUR_API_KEY" \
  -o video.mp4
```

## 错误码

| 状态码 | 说明 | 解决方案 |
|--------|------|----------|
| 200 | 成功 | - |
| 400 | 请求参数错误 | 检查必填参数 |
| 401 | 认证失败 | 检查 API Key 是否正确 |
| 403 | 无权限 | 检查余额或权限 |
| 404 | 资源不存在 | 检查 video_id |
| 429 | 请求过于频繁 | 增加轮询间隔 |
| 500 | 服务器错误 | 稍后重试 |

## 最佳实践

### 轮询策略

- **轮询间隔**: 30 秒
- **最长等待**: 10 分钟（600 秒）
- **典型耗时**: 3-5 分钟

### Prompt 编写建议

好的 Prompt 应包含：
- 场景描述
- 主体动作
- 镜头风格
- 艺术风格

**示例**:

```
电商宣传视频，时长10秒，竖屏9:16，720p，简约白色背景，
产品360°旋转展示，突出产品细节，电影级画质
```

## 项目集成说明

### Edge Function 配置

在 `supabase/functions/generate-sora-video/index.ts` 中：

```typescript
const SORA2_API_KEY = Deno.env.get('SORA_API_KEY') || 'YOUR_API_KEY';
```

### 环境变量设置

在 Supabase Dashboard 中设置：

1. 进入 Edge Functions → Secrets
2. 添加 `SORA_API_KEY` 变量
3. 值为有效的 API Key

### 部署命令

```bash
npx supabase functions deploy generate-sora-video --project-ref qbrxkgqhsfjgfatykyzv
npx supabase functions deploy query-sora-video --project-ref qbrxkgqhsfjgfatykyzv
```

## 当前问题排查

### 401 错误原因

1. **API Key 无效**: 当前硬编码的密钥可能已过期
2. **余额不足**: 账户余额用完
3. **Authorization 格式**: 尝试带/不带 `Bearer` 前缀

### 解决方案

1. 获取有效的 API Key
2. 在 Supabase Dashboard 中设置环境变量 `SORA_API_KEY`
3. 或直接更新 Edge Function 代码中的密钥
